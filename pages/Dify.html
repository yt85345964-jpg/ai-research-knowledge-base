Dify

一句话定位

Dify 是一个 LLMOps 平台，帮助开发者通过低代码方式持续定义和改进 AI 应用。名称源自 "define" + "modify"。

核心能力

Agent 四大支柱

能力 说明
自主规划 根据目标自动拆解任务步骤
工具调用 无缝集成搜索、计算器、数据库等外部 API
上下文记忆 短期对话 + 长期知识存储
人机协同 关键决策时主动请求人类介入

部署选项

方案 适用场景
Cloud（SaaS） 个人开发者、中小企业
社区版 有安全需求的团队
企业版 大型企业定制化部署

模型兼容性

支持 OpenAI GPT 系列、Anthropic Claude 系列及国内主流大模型，通过统一的模型供应商 API 配置接入。

与 WorkBuddy 的路线对比

维度 Dify WorkBuddy
定位 LLMOps 开发者平台 桌面 AI Agent 消费者平台
目标用户 开发者、技术团队 终端用户

综合判断：Dify 和 WorkBuddy 代表 Agent 产品化的两条路线，前者面向"会做 AI 应用的人"，后者面向"需要 AI 帮助的人"。两者共同验证了 Agent 能力需要封装和结构化的共识。

知识库整合能力

Dify 的知识库功能与本项目的 IMA → Wiki 管道有相似之处：上传文档 → 自动索引 → 智能检索。区别在于 Dify 是驱动 Agent 回答的 RAG 引擎，而本知识库是面向人类阅读的编译型知识体系。
