Problem Hunting

核心定义

**一句话定位**：Problem Hunting（问题狩猎）主张"找到问题占了整个事情的 80%"——Agent 让构建方案变简单后，真正的稀缺能力反转为**问题定义能力**。

概述

Problem Hunting 是 AI使用哲学 的四大支柱概念之一，由 Airtable CEO Howie Liu 提出。它刻画了 Agent 时代的价值反转：过去找问题容易、构建方案难（要写代码、雇人、花钱）；现在反转了——Agent 让构建方案变简单，真正的难度全在前端：你到底想解决什么问题？

实操建议：

  1. **列清单**：把反复在做 / 反复在烦 / 一直缺失的事全列出来。
  2. **不要上来就想"我要搭什么 Agent"**：先锚定问题，再谈方案。
  3. **问题清单 = Agent 部署计划**：清单里的每一项都是一个潜在的 Agent 落地点。

综合判断：本团队知识库的 AGENTS.md 设计哲学与 Problem Hunting 一致——先定义知识缺口（问题），再让 Agent 执行编译（方案）。详见 LLM Wiki。
