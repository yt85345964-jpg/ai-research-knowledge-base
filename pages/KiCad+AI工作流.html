KiCad + AI 工作流

核心定义

**一句话定位**："新手小白只靠和 AI 聊天就画出 PCB 原理图并导出 KiCad 工程文件"已完全可行——核心路径是 KiCad + AI Agent Skill/MCP/插件，2026 已进入"自然语言 → 真实电路"成熟期。

目标

让无深度 EDA 经验的工程师/新手，仅凭自然语言对话即可完成从需求分析、原理图/PCB 设计、验证到下单生产的完整 PCB 流程；缩短设计周期，降低 EDA 工具使用门槛，并实现一键 DFM 与下单。

适用场景

  • - 新手首次画 PCB、快速原型、测试治具、转接板等"先跑通再优化"的场景（Quilter.ai 适用）
  • - 日常可用的对话式 PCB 设计（华秋 KiCad Copilot 开箱即用）
  • - 程序员友好的参数化/代码化设计（SKiDL、atopile）
  • - 隐私/离线项目（Ollama + KIC-AI 本地大模型）
  • - 边界：端到端生成框架（CircuitLM 等）仍处于实验阶段，不直接用于生产

流程阶段 · 角色 · AI能力

阶段 负责角色 AI 能力 输入 → 输出 工具

关键步骤

  1. **需求（方案与选型）**：在 Copilot 面板输入自然语言（如"设计一个 5V 转 3.3V 的 LDO 电路"），AI 给出方案、核心器件和原理图；选中原理图/PCB 区域用 @上下文问答针对性提问；输入器件型号用 @查资料返回价格、库存、datasheet 链接。
  2. **设计（生成工程文件）**：
  3. - 华秋 KiCad Copilot：粘贴 Datasheet 引脚图/封装尺寸图，AI 识别并生成 KiCad 原理图符号或 PCB 封装（支持 BGA/QFN/QFP/SOP 等 IPC 标准封装）。
  4. - KiCad MCP：`claude mcp add --transport stdio kicad uvx kicad-mcp-server`，再描述需求（如"新建 50mm×50mm 的 LEDBoard，四角放 3mm 安装孔"），AI 调用工具落到 KiCad 工程。
  5. - atopile：`ato create project` / `ato create part -s` / `ato build`（每改一步 build 验证），纯 `ato build` 验证连接时不需要 KiCad。
  6. - SKiDL：用 Python 描述电路（如 `res = Part("Device","R",value="1k")`），生成网表/原理图。
  7. - 直接 Python 脚本：用 KiCad 内置 `pcbnew` API（PCB 编辑器自带 Python 控制台）放置器件、绘制走线。
  8. **验证**：AI 执行 ERC/DRC、DFM 预检，输出检查报告（KiCad MCP、Quilter.ai 物理驱动布局布线）。
  9. **下单**：检查完成后在华秋 KiCad / JLCPCB 一键 DFM + 下单，对接 LCSC 部件号选型与打样备料。
  10. **规则固化**：在项目根目录 `CLAUDE.md` 或自定义 Skill 中定义工作流、输出规范、验证门禁（build 必跑）与出错回滚策略；用 git 对 PCB 做版本控制。

效果指标

  • - 新手可画 PCB：新手只靠和 AI 聊天即可从模块设计到原理图生成、再到 KiCad 工程导出（2026 已日常可用）。
  • - 设计周期缩短：减少重复性手动操作，对话式/脚本化设计显著压缩画图时间。
  • - 一键 DFM + 下单：检查完可直接在华秋下单生产，打通设计到制造闭环。

异常处理

  • - 本地依赖缺失（KiCad MCP 依赖本地 KiCad 安装和 Python 环境）→ 改用华秋 KiCad Copilot（开箱即用）或 Ollama + KIC-AI（离线）方案。
  • - SKiDL / Python 代码错误：LLM 可能产生需修正的代码 → 需用户懂基本 Python 语法修正。
  • - 端到端生成框架（CircuitLM、直接 S-表达式）仍实验阶段 → 不用于正式生产，仅作研究。
  • - 设计意图不确定 → 用 Copilot 右键"猜测设计意图""归纳核心器件""未连接引脚检查"等预置菜单辅助。

实施方法与效果

  • - 实施方法：
  • - 新手路线：下载华秋 KiCad 发行版 → 安装 → 工具栏 Copilot 图标 → 输入需求 → AI 给方案+器件+原理图草稿 → 微调 → 一键 DFM → 一键下单。
  • - 进阶路线：安装 KiCad + Claude Code + KiCad MCP → 在 `CLAUDE.md` 定义设计工作流 → 自然语言描述需求 → MCP 工具调用 → 真实 KiCad 工程 → git 版本管理 + atopile 自动化验证。
  • - 极客路线：Ollama 本地大模型 + KIC-AI 插件，全程离线、零云端依赖，适合隐私项目/学习研究。
  • - 效果评估：
  • - "AI 画 PCB"已从"概念验证"进入"日常可用"（2026）。
  • - KiCad 生态 AI 集成是 2026 主旋律：华秋 Copilot、Quilter、KIC-AI、KiCad MCP、atopile——开源 EDA 的 AI 化比商业 EDA 更激进。
  • - 经验总结：
    • "AI Agent + 工具调用"（LLM 不直接写复杂代码，而是调用脚本/插件）是当前最优解；SKiDL 等代码生成中间件程序员友好但学习曲线陡。真正决定体验的是给 Claude Code 的规则文件（CLAUDE.md / Skill）。"自然语言 → 真实电路"是从手动画图 → 参数化脚本 → 自然语言对话的 EDA 范式跃迁，下一里程碑是"自然语言 → 完整产品"（含结构、PCB、固件）。
