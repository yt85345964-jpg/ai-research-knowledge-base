AI使用哲学

**一句话定义**：AI 时代拉开差距的不是工具使用能力，而是**Builder 心态**（主动折腾、驱动工具）和**判断力规模化**（把好判断施加到更多线程）——执行层交 Agent，判断层永远是人。

核心论点

单一来源观点：素材笔记/AI 理念/2026-07-03_顶尖1%的人用AI的方式 基于 Airtable CEO Howie Liu 在《Silicon Valley Girl》播客中的访谈。

AI 时代的竞争力结构正在发生根本性翻转：

维度 过去 现在（Agent 时代）
**核心竞争力** 执行能力（写代码、做PPT、搜信息） 判断力（发现问题、给精准反馈、做取舍）
**稀缺资源** 技术人才 问题定义能力
**工作角色** 个人贡献者（自己做事） 管理者（指挥 Agent 队伍）
**成本结构** 执行成本高，判断成本相对低 执行成本趋零，判断成本成为瓶颈

Builder 心态

"Builder"不是指会写代码，而是指愿意下场动手、驱动工具，而不是等着工具告诉你它还能做什么。

核心特征：

  • - **Tinker（折腾）**：没人知道"正确答案"，最有效的用法是在实践中发现的
  • - **不接受权威**："如果有人跑来跟你说，他知道每个场景该怎么搭完美的 Agent，这人是在忽悠你" — Howie Liu
  • - **类比**：像早期 PC 用户，不是研究怎么把牌拿更稳，而是想这副牌能打出多少组合
综合判断：Builder 心态与 Agent提示词设计模式 的「专家行为模式固化」并不矛盾。前者强调探索和试错，后者强调一旦发现有效模式就将其固化——两者构成「探索 → 固化 → 再探索」的循环。

判断力飞轮

Agent 让你可以同时跑 10-50 个并行任务（奢侈雇员），但每个任务的质量天花板取决于你的判断力：


Agent 跑更多实验 → 判断力积累更快 → 设计更好实验方向 → Agent 跑更多实验
       ↑                                                              |
       └──────────────────────── 飞轮 ────────────────────────────────┘

关键洞察：

  • - 判断力像肌肉，越练越强：以前一天做 3 个决策，现在可以做 30 个
  • - 失败成本趋零，但积累速度是 10 倍
  • - **Agent 越强，判断力的价值越大**——不是替代你，而是放大你

Problem Hunting（问题狩猎）

单一来源观点：Howie Liu 认为"找到问题占了整个事情的 80%"。

过去：找问题容易，构建方案难（要写代码、雇人、花钱）

现在：反转了。Agent 让构建方案变简单，真正的难度全在前端：你到底想解决什么问题？

实操建议：

  1. 列清单：把反复在做/反复在烦/一直缺失的事全列出来
  2. 不要上来就想"我要搭什么 Agent"
  3. 问题清单 = Agent 部署计划
综合判断：本团队知识库的 AGENTS.md 设计哲学与 Problem Hunting 一致——先定义知识缺口（问题），再让 Agent 执行编译（方案）。详见 LLM Wiki

奢侈雇员（Luxury Hires）

以前太多事不值得专门雇人做（旅行规划、行情追踪、营销实验……），Agent 让这些"奢侈岗位"变得可行：

  • - 设定偏好和规则，24 小时跑，不需要工资
  • - 商业意义：同时跑更多实验 → 更多击球机会 → 更多正确方向
  • - 这是 多Agent协作模式 的通俗表述：用多个 Agent 并行处理不同任务

入门路径

  1. 找一个好用的 Agent 工具上手，不要纠结选哪个
  2. 用低风险个人场景入手，不要上来就改造业务流程
  3. 把它当乐趣，不是任务——先作为消费者沉进去再上商业任务
综合判断：与 WorkBuddy 的产品设计理念一致——WorkBuddy 以「专家」封装复杂 Agent 能力，让用户无需懂提示词即可上手，降低入门门槛。
