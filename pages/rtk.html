rtk

高性能 CLI 代理，在命令输出到达 LLM 之前过滤压缩，降低 60-90% Token 消耗。

---

是什么

rtk 是一个单文件 Rust 二进制程序，零外部依赖。它作为 CLI 代理运行在 Shell 和 AI Agent 之间：拦截常见开发命令的输出，在传给 LLM 之前进行智能过滤、分组、截断和去重，将 Token 消耗降低 60-90%，额外开销 <10ms。

68.1k Stars（2026-07），支持 100+ 命令，兼容 Claude Code、Copilot、Cursor、Codex 等 14 种 AI 编程工具。

---

核心价值

给 AI 编程 Agent 的上下文窗口"瘦身"。Agent 执行 `git status`、`cargo test` 等命令时，大部分输出是 LLM 不需要的噪音。rtk 在中间层过滤掉冗余，让 Agent 用更少 Token 获得同样有效的信息。

一个 30 分钟 Claude Code 会话中，原始命令输出约 118,000 Token，经过 rtk 后降至约 23,900 Token（节省 80%）。

---

工作原理


Agent --命令--> Shell --原始输出--> LLM     ← 传统模式（大量 Token）
Agent --命令--> rtk --过滤--> LLM            ← rtk 模式（压缩输出）

四种策略：

  • - **智能过滤**：去除注释、空白、样板代码
  • - **分组聚合**：按目录聚合文件、按类型聚合错误
  • - **截断**：保留关键上下文、裁掉冗余行
  • - **去重**：重复日志行折叠为计数

---

关键特性

自动重写钩子（Auto-Rewrite Hook）

rtk 的杀手级功能。安装钩子后，所有 Bash 命令自动被重写为 rtk 等价命令：


git status        # → rtk git status（自动）
cargo test        # → rtk cargo test（自动）
ls -la            # → rtk ls（自动）

结果：100% rtk 采用率，Agent 和用户无需手动调用 rtk。

⚠️ 注意：钩子仅对 Bash 工具调用生效。Claude Code 的 Read、Grep、Glob 等内置工具不经过钩子，需要显式使用 `rtk read`、`rtk grep` 等命令或直接用 Shell 命令替代。

与其他工具的关系

  • - **Codex**：终端级 AI Agent，rtk 是其 Token 优化层，两者可叠加使用
  • - **WorkBuddy**：桌面 Agent 平台，同样面临上下文窗口效率问题，rtk 的思路可参考
  • - **Claude Code / Cursor / Copilot**：rtk 作为中间层透明介入，不需要修改这些工具本身

---

局限性

  1. **Windows 原生支持有限**：钩子依赖 Unix Shell，Windows 上自动重写不可用，只能退化为 CLAUDE.md 注入模式（AI 收到指令但命令不自动改写）。建议用 WSL。
  2. **不覆盖所有命令**：只支持预定义的命令集，自定义脚本需要自行配置
  3. **可能过滤掉有用信息**：偶尔会过于激进，导致 LLM 缺少关键上下文。可配置 `tee` 模式在失败时保存完整原始输出
