Harness(智能体执行壳)

核心定义

**一句话定位**：Harness 是包裹在大模型外面、让它成为一个可用智能体的完整基础设施。模型提供智能，Harness 提供手、眼、记忆和安全边界。
原文定义："An Agent Harness is the complete infrastructure that wraps around an LLM to make it a functional agent. The model provides intelligence; the harness provides hands, eyes, memory, and safety boundaries."

为什么重要

纯大模型只有「智能」，无法在真实世界中行动。Harness 补齐了从「能想」到「能做」的关键差距：

  • - **手**：Shell 命令、浏览器自动化(CDP/Playwright)、文件操作
  • - **眼**：读取文件、理解代码库、感知环境状态
  • - **记忆**：持久化上下文、跨会话知识沉淀
  • - **安全边界**：权限控制、沙箱隔离、审批机制

机制概述

Harness 与编排层的分工


编排层(如 OpenClaw Gateway)
  → 负责：何时做、给谁做、做完谁审批、全渠道接入、心跳调度
执行层(Harness)
  → 负责：具体怎么写代码、理解代码库、多文件修改、跑测试、开 PR

两者通过 ACP(Agent Client Protocol) 桥接，实现编排与执行解耦——OpenClaw 不需要重新实现代码理解能力，Claude Code 也不需要关心消息从哪个渠道来。

编程 Harness 的关键能力

  1. **会话可恢复**：重启/换设备不丢进度（resumeSessionId + session/load）
  2. **独立 worktree 隔离**：多任务并行不冲突
  3. **分级审批**：直接执行 / 委托编排层复核 / 人工审批 / 自动开 PR

与其他概念的关系

  • - **Harness vs Agent Framework**：Agent Framework（如 OpenClaw）是「编排壳」，管调度、渠道、心跳；Harness 是「执行壳」，管具体任务执行；一个 Framework 可以桥接多个 Harness（ACP 协议）。
  • - **Harness vs MCP**：MCP 解决「智能体如何发现和调用外部工具/数据源」；Harness 解决「智能体如何在真实环境中行动」；MCP 是 Harness 的「工具接入协议层」，Harness 是更完整的执行基础设施。

安全考量

Harness 拥有对宿主环境的操作权限（Shell 命令、文件读写、网络通信），是 Simon Willison「致命三角」的核心风险点之一：

  • - **访问私有数据**：Harness 可以读取代码、凭据、文件
  • - **对外通信能力**：Harness 可以执行网络命令、调 API
  • - **处理不可信内容**：Harness 会处理外部输入、第三方技能指令

风险缓解：环境隔离（容器/受限用户）、权限最小化、高风险动作强制人工审批。
