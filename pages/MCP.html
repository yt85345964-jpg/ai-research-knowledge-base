MCP

核心定义

**一句话定位**：MCP（Model Context Protocol，模型上下文协议）是由 Anthropic 于 2024-11-25 提出的**开放标准协议**，为大型语言模型（LLM）与外部数据源、工具之间提供统一的通信接口，常被类比为「AI 应用的 USB-C 接口」。
原文定位（Anthropic）："Think of MCP like a USB-C port for AI applications." MCP 让 Claude、ChatGPT 等 AI 应用能连接数据源（本地文件、数据库）与工具（搜索引擎、计算器）及工作流（专用提示词），从而访问关键信息并执行任务。（来源：modelcontextprotocol.io）

为什么重要

MCP 出现前，大模型与外部交互存在物理隔离、协议碎片化、时效性缺失三大问题——每个数据源/工具都要写一套定制集成，形成「N×M」的对接地狱。MCP 把它收敛为「N+M」：

  • - **开发者**：降低构建/集成 AI 应用或 Agent 的复杂度
  • - **AI 应用/Agent**：获得统一的数据源与工具生态，能力增强
  • - **终端用户**：得到能访问你的数据、在必要时代表你行动的更强 AI

关键机制

机制概述

核心架构（客户端-服务器）：MCP 遵循客户端-服务器（client-server）架构，三个核心组件：

组件 角色 说明
**MCP Host（主机）** 发起请求的 LLM 应用 Claude Desktop、Cursor、IDE 等；负责初始化/管理客户端、用户授权、上下文聚合
**MCP Client（客户端）** 宿主内运行，与 Server 1:1 连接 AI 助手与 Server 间的通信桥梁；消息路由、能力管理、协议协商、订阅管理
**MCP Server（服务器）** 轻量级程序，暴露特定功能 每个 Server 专注一种资源/功能（文件系统、数据库、Web 搜索等），模块化保证可扩展

三种原语（Primitives）：MCP Server 通过三类能力向 AI 提供上下文与功能：

  • - **Tools（工具）**：可执行函数（如发邮件、调 API、查数据库）——AI 可主动调用
  • - **Resources（资源）**：静态/只读数据源（文件、数据库记录）——为模型提供实时或历史数据
  • - **Prompts（提示词）**：预定义任务模板——标准化 LLM 交互流程，保证连贯性

通信机制：

  • - **协议**：JSON-RPC 2.0
  • - **传输方式**：stdio（标准输入输出，本地进程间通信）、SSE（Server-Sent Events，基于 HTTP 长连接，支持异步数据流，远程）、Streamable HTTP（较新的远程传输方式）

与 Harness / Agent 的关系：MCP 解决「智能体如何发现和调用外部工具/数据源」，Harness 解决「智能体如何在真实环境中行动」——两者是不同层次：

  • - MCP = Harness 的「工具接入协议层」
  • - Harness(智能体执行壳) = 更完整的执行基础设施（手、眼、记忆、安全边界）

例如 OpenClaw 通过 MCPorter 接入 MCP 生态，让编排壳能调用外部工具；又通过 ACP（Agent Client Protocol）桥接编程 Harness（Claude Code / Codex）。MCP 与 ACP 分工不同：MCP 管「工具/数据接入」，ACP 管「编排层与执行层的会话桥接」。

生态与采用：

  • - **2024-11-25**：Anthropic 正式发布
  • - **2025-03**：OpenAI 宣布 Agent SDK 支持 MCP，ChatGPT 桌面应用及 Responses API 接入
  • - **2025-12**：Linux 基金会成立 Agentic AI Foundation（AAIF），MCP 正式移交该基金会管理
  • - 截至 2025 年，已有 1000+ 社区 Server 和数千个集成应用；国内百度地图、高德、腾讯位置服务等已推出 MCP Server
  • - 支付宝（2025-04）推出国内首个支付领域 MCP Server；百度发布 MCP store

安全考量：MCP Server 拥有对宿主环境的操作能力（文件读写、API 调用、网络通信），存在与 Harness 类似的「致命三角」风险（访问私有数据 + 处理不可信内容 + 对外通信）。缓解原则：

  • - Server 最小权限、沙箱隔离
  • - 工具调用（尤其写操作）强制人工审批
  • - 供应链管控：仅安装可信 Server，审查第三方 Server 实现
  • - 所有调用可审计、可回滚

关键公式 / 伪代码

Client 工作流：


从 Server 获取工具列表
  → 将用户查询 + 工具描述经 function calling 发给 LLM
  → LLM 决定调用哪些工具
  → 经 Server 执行
  → 结果回传 LLM
  → 生成自然语言响应

适用场景

  • - 让 AI 应用 / Agent 统一接入数据源（本地文件、数据库）与工具（搜索引擎、计算器）及工作流（专用提示词）
  • - 降低构建 / 集成 AI 应用或 Agent 的复杂度（把「N×M」对接地狱收敛为「N+M」）
