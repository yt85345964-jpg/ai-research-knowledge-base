YC内部AI实操：如何让AI从助手进化到"大脑"

一句话总结

YC 内部 AI 系统的真正护城河不是模型，而是一条有先后顺序的路径：数据集中 → 工具注册表 → 技能自我进化。顺序反了，每一步都会卡死。

核心架构：三件套

1. 数据集中（Context Layer）

  • - YC 所有自研软件跑在同一个 Postgres 数据库上：公司表、创始人表、财务表、CRM笔记
  • - Agent 获得了对业务数据的完整只读查询权限（最初由工程师 Jared "违规"半夜偷偷推送）
  • - 效果：杰文斯悖论——降低查询成本反而大幅增加提问数量和复杂度

2. 工具注册表（Tool Registry）

  • - 从 20 个工具起步，各团队持续添加，现已超过 350 个
  • - 每个团队加自己的工具：财务团队记账、合伙人管理办公时间、活动团队管理活动
  • - 工具注册表 = 让 Agent 从"能聊天的助手"变成"能干活的同事"的关键
  • - 不仅内部 Agent 可用，Claude Code 也可以调用

3. 技能系统（Skill System）

  • - 技能 = 对工具的更高层封装，可复用的能力单元
  • - Skillify 元技能：一键固化新操作为可复用技能
  • - check resolvable 元技能：DRY + MECE，去重+完整性检查
  • - **Dream Cycle（夜间自我改进）**：每晚自动读取当天所有对话，找改进点，次日技能提示词自动优化

关键案例：两句话描述技能进化

  1. 合伙人 Tom 手写提示词 → 第一版技能
  2. 多位合伙人在 group office hours 让创始人练习 → 隐性经验显性化
  3. 会议转录交给 Agent → 根据学到的经验改进技能
  4. 改进后的技能**超越任何一位合伙人单独的水平**

→ 这就是组织超级智能的发生方式：智能来自沉淀人类专家散落在对话中的隐性经验

关键设计决策

默认公开（Default Transparency）

  • - 所有 Agent 对话对 YC 全职员工全局可见，广播到 Slack 频道
  • - 三个作用：知识传播、社会性安全控制（替代技术访问控制）、降低新员工门槛
  • - 新员工通过阅读他人对话完成"学徒制"，上手速度从6个月大幅缩短

默认信任 + 平等主义

  • - AI 能力向组织中每个人开放，不只领导层
  • - 信息不按权限层级锁定
  • - **大多数组织是 command and control，这与 Agent 需要广泛上下文的特性根本矛盾**

"无马马车"隐喻

  • - 大量 AI 软件是"确定性软件包裹一小块 AI"（如 Gmail AI 写邮件）
  • - 正确方向是**Agent 包裹确定性工具**
  • - 核心问题：控制权应从开发者转移到用户

企业三步走路径

步骤 做什么 关键点
① 数据集中 打破数据壁垒，把关键业务数据收到一个地方 哪怕只是共享数据库
② 工具注册表 把团队重复做的事变成 AI 可调用工具 **从最痛场景开始**（YC 从财务开始）
③ 技能自进化 工具之上建技能层，接入夜间自我改进 让系统每天比昨天聪明一点

顺序不能反：没有集中数据 → 工具调不到东西；没有稳定工具 → 技能是空中楼阁；没有沉淀对话 → 自我改进没原材料

个人三步走路径

  1. **数据集中**：笔记、文档、对话收到一个地方
  2. **记录一切**：会议录音、决策过程、与AI对话——这是迭代原材料
  3. **从用户变训练师**：不是让AI帮你做事，是让AI越来越懂你怎么做事

窗口期论

  • - 现在花 $10-100K/年做的事，两年后可能只要几百块
  • - 但几百块的能力和十万美元的能力之间，差的就是这条路径
  • - 模型会越来越便宜，不确定的是：你有没有提前把数据、工具、判断力沉淀成会自我进化的系统
