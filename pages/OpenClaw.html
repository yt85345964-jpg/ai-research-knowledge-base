OpenClaw

核心定义

**一句话定位**：OpenClaw 是一个开源(MIT)、自托管的「个人 AI 助手」框架——它本身不是大模型，而是模型无关的编排壳，通过 Markdown 配置文件定义行为，支持 40+ 消息渠道接入，内置技能系统、心跳调度和 MCP 集成。

核心功能

核心组件（五件套）

组件 作用 关键文件/机制
Gateway（网关） 常驻进程，统一控制平面 JSON-RPC/WebSocket，所有渠道/会话/配置统一通信
Brain（推理引擎） 底层大模型路由 模型无关，可接 Claude/GPT/Gemini/DeepSeek/Kimi/GLM/Ollama
Hands（执行环境） Shell/浏览器/文件/智能家居操作 由 Skills 与内置工具提供
Memory（记忆系统） 零运维本地持久化记忆 Markdown 文件 + SQLite FTS5，无云依赖
Heartbeat（心跳） 默认 30 分钟自主循环 HEARTBEAT.md 定义，是「主动而非被动」的关键机制

关键能力

  • - **全渠道接入**：飞书、企业微信、Slack、Telegram、Discord、Web UI、CLI 等 40+ 渠道
  • - **技能系统(Skills)**：SKILL.md 格式，渐进式披露，模型按需翻阅而非一次性塞入上下文；三种来源：人工编写、ClawHub(社区市场)、智能体自主生成
  • - **心跳调度(Heartbeat)**：周期性自主循环，支撑晨报、巡检等「主动性」任务
  • - **MCP 集成(MCPorter)**：通过标准协议接入外部工具和数据源
  • - **ACP 桥接**：通过 Agent Client Protocol 接入外部编程 Harness（Claude Code、Codex CLI 等）

与 Harness 的关系

OpenClaw 内置轻量 Harness（代号 Pi），同时通过 ACP 把重量级编程 Harness 接入为后台可恢复会话：


编排层(OpenClaw) ←ACP→ 执行层(Claude Code/Codex CLI)
  负责：何时做、给谁做     负责：具体怎么写代码

这是 OpenClaw 生态已验证的成熟模式（有 openclaw-code-agent、claw-orchestrator 等社区插件）。详见 Harness(智能体执行壳)。

设计原则

OpenClaw+Harness 架构方案的五条设计原则，对任何 AI Agent 系统都有参考价值：

  1. **本地优先/数据主权** — 核心数据本地 Markdown+SQLite，云端以可插拔增强层接入
  2. **渐进式自主权** — 从「只建议不执行」起步，积累验证后才放开
  3. **人在回路** — 高风险动作设强制审批点
  4. **可审计可回滚** — 纯文本配置天然适合 Git 版本化
  5. **关注点分离** — 编排/执行/工具/存储各自独立可替换
综合判断：这些原则与本知识库 AGENTS.md 的设计理念高度一致——本地 Markdown 优先、知识可追溯、人审核关键变更。

⚠️ 安全风险（重要）

单一来源观点：以下安全事件均来自多家安全机构交叉验证，但风险严重程度需要结合实际部署环境评估。
事件 详情 来源
CVE-2026-25253 (CVSS 8.8) 跨权限边界资源传递不当 安全研究团队 + 官方修复
ClawHavoc 供应链投毒 300+ 恶意技能包，生态约 10-20% 恶意 Koi Security、Bitdefender 等
ToxicSkills 审计 约 36% 提示注入特征 Snyk 专项报告
默认配置暴露公网 约 13.5 万暴露实例 安全评级机构扫描
恶意软件捆绑 部分捆绑 macOS infostealer 安全厂商威胁情报

核心教训：Agent Skill 运行时拥有宿主完全权限，不像传统软件包有沙箱隔离。安装恶意技能 ≈ 把对应权限交给未知第三方。

致命三角(Simon Willison)：访问私有数据 + 处理不可信内容 + 具备对外通信能力 → 风险指数级放大。

加固建议（10 条）：

  1. 网络隔离：Gateway 仅监听本地回环/内网
  2. 账号隔离：系统账号与个人主账号分离
  3. 环境隔离：容器/受限用户权限
  4. 技能供应链管控：安装前扫描+信誉检查
  5. 自生成技能同等审查
  6. 记忆写入治理：高风险规则不自动生效
  7. 持续完整性监控：校验和+漂移检测
  8. 审计可回滚：所有变更纳入 Git
  9. 高风险动作强制人工审批
  10. 及时更新：关注安全公告，第一时间升级
