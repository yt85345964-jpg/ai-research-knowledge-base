嘉立创 EDA AI 操控

核心定义

**一句话定位**：嘉立创 EDA 专业版已可通过**官方扩展 / 第三方 Skill / Agent Connector** 三种路径实现"AI 直接操控编辑器画原理图与 PCB"，目前**已证明路径可行但效果一般**——复杂场景仍需工程师介入。

核心功能

三套方案对比

  • - **方案一：社区 Skill（easyeda-api / easyeda-agent）⭐⭐⭐⭐**
  • - **架构**：Skill 描述工作流 → Go CLI/Daemon (端口 49620-49629) → WebSocket → .eext Connector → eda.* API
  • - **安装**：`curl -fsSL https://raw.githubusercontent.com/zhoushoujianwork/easyeda-agent/main/install.sh | sh`；在嘉立创 EDA 扩展管理器安装 EasyEDA Agent Connector 扩展（github.com/zhoushoujianwork/easyeda-agent，Apache-2.0）；`easyeda daemon` 启动守护进程；`easyeda health` 验证（→ daemon: online, connector: connected）
  • - **AI 控制命令示例**（与 Hermes / Claude Code / Codex 配合）：
    • 
    easyeda sch search --keyword STM32H753
    easyeda sch place --lib-uuid xxx --designator U1 --x 3000 --y 3000
    easyeda sch connect-pin --designator U1 --pin PA9 --net USART1_TX
    easyeda sch power-flag --net VDD_3V3
    easyeda sch ground-flag --net GND
    easyeda sch drc
    easyeda sch export netlist
    easyeda sch export bom --format xlsx
  • - **方案二：官方 eext-run-api-gateway ⭐⭐⭐⭐**
  • - **架构**：AI 编程工具 (Cursor/Claude Code/Hermes) ← WebSocket → eext-run-api-gateway → LCEDA 编辑器
  • - 在嘉立创 EDA Pro → 扩展 → 扩展广场 搜索 `eext-run-api-gateway` 安装；网关默认端口 9527；通过 WebSocket 调用 LCEDA 99% 功能
  • - **Python 客户端示例**：`await lceda("run_drc", {"type": "sch"})`；`await lceda("export_bom", {"format": "csv"})`
  • - **方案三：自定义扩展文件桥接 ⭐⭐**
  • - 传统方式：Hermes Agent 将网表写入 JSON 文件；LCEDA 扩展轮询读取后执行；通用性差，仅作过渡方案

Claude Code 对接示例

  1. 安装 easyeda-api skill（输入给 Claude Code：`https://github.com/easyeda/easyeda-api-skill 帮我安装这个 skill`，或自行下载放在 skill 目录）
  2. 嘉立创 EDA 侧安装扩展（高级 → 扩展管理器 → 搜 "run" → 安装 → 勾选三个选项）
  3. `claude code` → `EDA 启动！` → 建立 WebSocket 连接，即可让 AI 画原理图

三大应用场景

  1. **自动画原理图**：减少重复性手动操作，缩短设计周期；一致性保证，避免人为错误；知识沉淀为可复用自动化脚本；让工程师专注创造性设计
  2. **嵌入式开发协同**（配合 Codex / Claude Code）：自动建库（easyeda-api 提供 KiCad 库 → 立创商城映射）；一键导出 BOM（与立创商城库存联动）；DFM 预检（与立创 PCB/SMT 工艺匹配）
  3. **平衡车 / 自动驾驶项目**：嘉立创 EDA 是嵌入式实战跳板（电源管理、MCU、运动传感器、电机驱动、无线通信、传感器接口等核心模块），配合 SailWind 等 AI EDA 工具形成完整链路

适用场景

适用 不适用
简单原理图模块搭建、基础器件放置、单步连线 复杂 PCB 布局、高速信号规则布线、多层板电源平面优化
BOM/Netlist 导出、DRC 检查 对精度要求极高的关键参数（阻抗/差分/等长）调整
工程师主导 + AI 辅助的嵌入式实战项目 完全无人介入的端到端 PCB 生成
实际能力边界：✅ 适用：简单原理图模块搭建、基础器件放置、单步连线、BOM/Netlist 导出；❌ 不足：复杂 PCB 布局、高速信号规则布线、多层板电源平面优化；⚠️ 仍需工程师：DRC 误报甄别、关键参数调整。

团队评估

关键洞察：
1. **AI 操控 EDA 已"证明可行"**：从代码/自然语言到真实编辑器里的 typed action，已走过 0→1 阶段。瓶颈在 1→100 的工程化细节。
2. **三方案生态井喷**：官方 gateway + 社区 Skill + Agent Connector，说明这是行业级共识，嘉立创 EDA 正主动接入 AI Agent 生态。
3. **计算密集型场景**：AI 协作需要 32GB+ 内存，复杂工程 64GB+；Node.js Bridge Server 默认堆内存 1.4GB 不足，需要 `--max-old-space-size=4096`。
4. **与 KiCad 形成两条路径**：嘉立创 EDA（国产 + 商业生态）vs KiCad（开源 + 全球生态），AI 化路径趋同但工具栈不同。
