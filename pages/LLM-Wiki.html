LLM Wiki

核心定义

**一句话定位**：LLM Wiki 是一种「编译式知识库」范式：让 LLM 在用户和原始文档之间扮演**知识编译器**，把原始资料提前编译成结构化、相互链接、持续维护的 Markdown Wiki，而不是每次提问都临时从原文检索。

综合判断：本团队知识库（AGENTS.md 设计的 `原始资料/` → `知识库/` → `AGENTS.md/方案.md` 三层结构）正是 LLM Wiki 范式的直接实践。详见文末「本知识库即 LLM Wiki 实践」。

概述

覆盖什么范围？为什么值得单独成页？

LLM Wiki 把知识库从「检索式」翻转成「编译式」：让 LLM 在摄入阶段就主动提取、整合、更新结构化 Wiki，使知识沉淀为可复用、可溯源、可迭代的「内存」，而非每次查询都从原始「硬盘」临时检索。

核心思想：编译 vs 检索（Karpathy 指出传统 RAG 的致命缺陷："There's no accumulation."（没有积累），来源：Karpathy，LLM Wiki 原始提出材料）：

范式 核心动作 类比 知识是否沉淀
**传统 RAG（检索式）** 查询时从原始文档检索→理解→综合 解释型语言：每次运行都重新编译 ❌ 不沉淀，重复计算
**LLM Wiki（编译式）** 摄入时 LLM 主动提取、整合、更新 Wiki 编译型语言：提前编译成中间表示 ✅ 一次编译，多次复用

关键差异：RAG 像「每次考试都重新翻书」；LLM Wiki 像「提前上过课，知识点已经记住，考试直接答」。

三层架构：


your-wiki/
├── raw/      # 第一层：原始资料区（不可变，LLM 只读不写，唯一事实来源）
├── wiki/     # 第二层：LLM 编译后的知识库（核心，Markdown 文件集合）
└── schema/   # 第三层：系统配置与 AI 行为规则（本库的 AGENTS.md/方案.md）
  • - **raw/**：论文、文章、笔记、图片、代码库等；LLM 只读不写，保证来源可追溯、可被重建
  • - **wiki/**：由 LLM 全权维护的 Markdown 集合，含百科式文章、概念摘要、交叉链接、矛盾标注、综合分析
  • - **schema/**：给 AI 的行动指南，定义摄入、页面生成、知识整合、问答规则——通过这份文件把 LLM 从通用聊天机器人变成「有纪律的 Wiki 维护者」

综合判断：三层职责清晰分离是 LLM Wiki 站得住脚的根本原因——存储、组织、展示互不纠缠，每层可独立替换（任何 Markdown 阅读器替代 Obsidian，Claude Code 可换 Codex，schema 随领域演进）。

关键工作流：

  1. **摄入（Ingest）**：将新资料放入 `raw/` 目录
  2. **编译（Compile）**：LLM 阅读资料，提取关键信息，创建/更新 Wiki 页面，维护交叉链接，标记矛盾点
  3. **查询（Query）**：基于已编译的 Wiki 回答，无需再访问原始资料
  4. **迭代（Iterate）**：每次查询和新资料摄入都进一步丰富和完善知识库

编译环节的特殊之处在于「通读整个现有 Wiki」：一个新来源可能触及 10–15 个已有页面，模型会更新所有相关内容——每个新输入都被整合进现有知识结构，而非堆叠在上面。

为什么有效：知识复利

Wiki 不是静态存档，而是随每个新文档持续演化：每读一篇新资料，更新相关概念解释、补充交叉引用、标记与旧知识的矛盾；积累越久，Wiki 越丰富；已有关联越完整，吸收新知识的成本越低。第 100 个来源的处理，建立在已蒸馏前 99 个来源知识的 Wiki 之上，不会从零起步——这就是「复利效应」的实质。

主流观点 / 方法对比

观点 / 方法 代表来源 核心思想 优势 劣势
传统 RAG（检索式） Karpathy 等 查询时从原始文档检索→理解→综合 覆盖面广、速度快、无需预综合 没有积累，重复计算
LLM Wiki（编译式） Karpathy 等 摄入时 LLM 主动提取、整合、更新 Wiki 一次编译多次复用、质量高、可溯源 需持续维护，未精细阅读文档覆盖弱

与 RAG 的关系（互补而非取代）：生产环境两者常互补——先查 Wiki（精确、综合好、人类可读），再查向量数据库（大规模、语义检索），合并结果生成回答。

知识类型 推荐方案 原因
精心阅读过的文档 LLM Wiki 已预综合，质量高
大量未精细阅读的文档 RAG 覆盖面广
需要跨文档综合的分析 LLM Wiki 综合已预编译在 Wiki 中
简单事实检索 RAG 速度快，不需要综合

LLM Wiki 层 本知识库对应
raw/（不可变原始资料） `原始资料/`
