WorkBuddy

核心定义

**一句话定位**：WorkBuddy 是腾讯推出的桌面 AI Agent 平台，将底层 Agent 能力封装为「专家」和「专家团」，让用户无需了解提示词工程即可使用专业 AI 能力。

核心功能

专家（Single Expert）

  • - 封装好的单个 Agent，内置角色定义、工作流程、交付模板
  • - 用户选择专家即可开始工作，无需编写提示词
  • - 示例：用户体验架构师、长文档手稿专家

专家团（Expert Squad）

  • - 封装好的多 Agent 协作流程
  • - 采用「调度-执行」模式（编排者模式）
  • - 主理人负责拆需求、配成员、看进度、收产出
  • - 示例：AI 内容创作专家团

技术特点

  • - **专家提示词设计**：结构 = 普通Agent + 领域方法论 + 交付模板 + 工作流约束
  • - 角色覆盖：Role Override 机制重置模型身份
  • - 工作流程：明确 SOP 约束 Agent 行为
  • - 交付标准：定义成功指标和交付物模板
  • - 沟通风格：固化专业行为模式
  • - 详见 Agent提示词设计模式
  • - **多Agent协作机制**：详见 多Agent协作模式
  • - 任务分配预检（防止误派）
  • - 预设 Workflow（固化协作流程）
  • - 消息中转模式（避免上下文污染）
  • - 异常处理和交付检查

自定义模型接入

WorkBuddy 支持通过 OpenAI 兼容协议接入自定义模型，不限于内置模型商：

  • - **配置方式**：设置 → 模型 → 新增模型 → 自定义；填写接口地址（需兼容 OpenAI API 协议）、API Key、模型名称、上下文窗口；保存后在对话界面底部切换使用
  • - **企业场景**：支持通过企业内部 AI API网关 接入，员工无需各自申请模型 API Key，由网关统一管理令牌和额度。来源：素材笔记/AI流程/Workbuddy配置公司中转站操作手册 中记录了通过华诺星空 AI 网关接入 deepseek-v4-pro 的完整配置流程。

团队评估

WorkBuddy 的专家系统代表了 Agent 产品化的一个方向：将提示词工程的最佳实践固化为可复用的专家角色。对我们的 AI 研究知识库来说，其 AGENTS.md 的设计思路与此高度一致。
