WorkBuddy

一句话定位

WorkBuddy 是腾讯推出的桌面 AI Agent 平台，将底层 Agent 能力封装为「专家」和「专家团」，让用户无需了解提示词工程即可使用专业 AI 能力。

核心功能

专家（Single Expert）

  • - 封装好的单个 Agent，内置角色定义、工作流程、交付模板
  • - 用户选择专家即可开始工作，无需编写提示词
  • - 示例：用户体验架构师、长文档手稿专家

专家团（Expert Squad）

  • - 封装好的多 Agent 协作流程
  • - 采用「调度-执行」模式（编排者模式）
  • - 主理人负责拆需求、配成员、看进度、收产出
  • - 示例：AI 内容创作专家团

技术特点

专家提示词设计

WorkBuddy 专家的提示词结构 = 普通Agent + 领域方法论 + 交付模板 + 工作流约束

核心维度：

  1. **角色覆盖**：Role Override 机制重置模型身份
  2. **工作流程**：明确 SOP 约束 Agent 行为
  3. **交付标准**：定义成功指标和交付物模板
  4. **沟通风格**：固化专业行为模式

详见 Agent提示词设计模式

多Agent协作机制

详见 多Agent协作模式

关键设计：

  • - 任务分配预检（防止误派）
  • - 预设 Workflow（固化协作流程）
  • - 消息中转模式（避免上下文污染）
  • - 异常处理和交付检查

适用场景

适用 不适用

跨领域协作的复杂项目 对实时性要求极高的场景
内容创作全链路 需要私有化部署的场景

自定义模型接入

WorkBuddy 支持通过 OpenAI 兼容协议接入自定义模型，不限于内置模型商：

配置方式

  1. 设置 → 模型 → 新增模型 → 自定义
  2. 填写接口地址（需兼容 OpenAI API 协议）、API Key、模型名称、上下文窗口
  3. 保存后在对话界面底部切换使用

企业场景

支持通过企业内部 AI API网关 接入，员工无需各自申请模型 API Key，由网关统一管理令牌和额度。
