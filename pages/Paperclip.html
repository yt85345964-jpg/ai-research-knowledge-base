Paperclip

核心定义

**一句话定位**：Paperclip 是 2026 年 3 月上线的开源（MIT）「AI Agent 公司管理系统」——把 Agent 当员工管理。GitHub 7.2 万星、一行命令开箱即用。**「如果 OpenClaw 是一个员工，Paperclip 就是整个公司」**。

核心功能

核心理念

2025 年是 AI Agent 的个体能力爆发年——Claude Code 能写生产级代码、Codex 能独立交付模块、OpenClaw 能连续运行数小时。**真正的瓶颈不是能力，是规模化的协调**。

痛点场景：

  • - 同时开 20 个 Claude Code 窗口，记不清谁在干什么
  • - 重启后所有工作上下文消失
  • - 一个失控循环烧掉几百美元 token
  • - 花 15 分钟找上下文提醒 Agent 它在做什么

六大核心能力

  1. **组织架构**：Agent 有职位、头衔、汇报线、权限、预算；严格的树状层级（CEO → CTO → 工程师）；上报与任务委派沿组织架构自动流动
  2. **目标对齐**：每个任务都追溯到公司使命；Agent 知道做什么，更知道为什么做
  3. **预算控制**：每个 Agent 有月度预算，到限额自动停止；80% 警告 / 100% 自动暂停 / 董事会可 override；防止"跑一晚烧几百刀"的事故
  4. **任务管理**：原子化任务签出：不重复执行、不丢失上下文；支持阻塞依赖、评论、附件；状态机驱动
  5. **心跳调度（核心创新）**：Agent 按计划醒来 → 检查工作 → 执行任务 → 休眠；间隔可配（15 分钟 ~ 数小时）；事件触发：@-mention 或直接任务分配时立即唤醒；解决 Agent 无状态问题
  6. **治理与审批**：董事会审批工作流（结构性变更需批准）；执行策略：哪些可自主、哪些需人工 sign-off；版本化配置 + rollback 能力
  7. **完整审计**：每次对话可追溯、每个决策有解释；完整工具调用追踪 + 不可变审计日志
  8. **多公司隔离**：一个部署可跑无数公司；数据完全隔离（多租户）

支持的 Agent 类型（BYO-Agent）

  • - ✅ **Codex**
  • - ✅ **Claude Code**
  • - ✅ **Cursor / Gemini / bash 等 CLI Agent**
  • - ✅ **OpenClaw**（HTTP/Webhook Bot）
  • - ✅ Nous Research Hermes（via hermes-paperclip-adapter）
  • - ✅ Pi / OpenCode
  • - ✅ 任何能接收心跳的 Agent
"Bring Your Own Agent" 哲学——任何能接收心跳信号的事物都可以被"雇进"组织架构。

适用场景

  • - ✅ 团队 Agent 数量从「几个」到「几十个」的过渡阶段
  • - ✅ 需要预算控制、审计追踪、董事会审批的中大型组织
  • - ✅ 已经在用 Claude Code / Codex / OpenClaw 想统一管理

团队评估

关键洞察：
1. **"AI Agent 协调层"是新蓝海**：当 Agent 数量突破单兵作战阈值，"管理 Agent 的系统"成为新刚需。Paperclip 4 个月 7 万星印证了这一点。
2. **"Bring Your Own Agent" 是正确抽象**：Paperclip 不做模型、不做 Agent——只做协调。
3. **「心跳 + 预算 + 审计」三件套是 Agent Ops 标配**：缺一不可。
4. **从"Copilot"到"Co-Org"**：AI 进化三阶段——Copilot（辅助）、Co-Worker（同事）、Co-Org（组织成员）。Paperclip 是 Co-Org 阶段的开源代表。详见 AI原生组织。
