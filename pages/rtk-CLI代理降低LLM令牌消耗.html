rtk — 高性能 CLI 代理，降低 LLM Token 消耗 60-90%

来源：GitHub 仓库 rtk-ai/rtk | 68.1k Stars | 4.2k Forks | Rust 93.1%
版本：v0.43.0 | 许可证：Apache-2.0

---

一句话总结

rtk 是一个单文件 Rust 二进制 CLI 代理，在命令输出到达 LLM 上下文之前进行过滤和压缩，使常见开发命令的 Token 消耗降低 60-90%，开销 <10ms。支持 100+ 命令，兼容 14 种 AI 编程工具。

---

核心机制


没有 rtk：                       有 rtk：

Claude --git status--> shell --> git        Claude --git status--> rtk --> git
  ^                              |              ^                    |        |
  |   ~2000 tokens（原始输出）    |              |   ~200 tokens      | 过滤   |
  +------------------------------+              +---（过滤后）---------+--------+

四种策略按命令类型组合应用：

策略 说明
**智能过滤** 去除噪音（注释、空白、样板代码）
**分组聚合** 同类项合并（按目录聚合文件、按类型聚合错误）
**截断** 保留相关上下文，裁掉冗余
**去重** 重复日志行折叠为计数

支持的命令分类

文件操作

`rtk ls`、`rtk read`（智能文件读取）、`rtk smart`（2行启发式摘要）、`rtk find`、`rtk grep`

Git

`rtk git status`（紧凑状态）、`rtk git log`（单行提交）、`rtk git diff`（压缩差异）、`rtk git add/commit/push/pull`

测试运行器

`rtk jest`、`rtk vitest`、`rtk pytest`（-90%）、`rtk go test`（-90%）、`rtk cargo test`（-90%）、`rtk rspec`、`rtk playwright test`

构建与检查

`rtk lint`（ESLint 按规则/文件分组）、`rtk tsc`、`rtk cargo build`（-80%）、`rtk ruff check`（-80%）

容器与基础设施

`rtk docker ps/logs/compose`、`rtk kubectl pods/logs`、`rtk pulumi preview/up`

数据分析

`rtk json`（无值的结构）、`rtk env`（过滤环境变量）、`rtk log`（去重日志）、`rtk curl`

---

自动重写钩子（Auto-Rewrite Hook）

核心用法。钩子透明拦截 Bash 命令调用，自动重写为 rtk 等价命令。结果：跨所有对话和子 Agent 达到 100% rtk 采用率，零 Token 开销。

⚠️ 注意：仅对 Bash 工具调用生效。Claude Code 内置工具（Read、Grep、Glob）不经过 Bash 钩子，不会被自动重写。

安装


# Homebrew（推荐）
brew install rtk

# 快速安装（Linux/macOS）
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh

# Cargo
cargo install --git https://github.com/rtk-ai/rtk

# Windows：下载发布包解压，放入 PATH

隐私与遥测

  • - 遥测**默认关闭**，需显式 opt-in
  • - 收集：匿名使用统计（命令数量、Token 节省量、Agent 类型分布）
  • - **不收集**：源代码、文件路径、命令参数、密钥、环境变量、个人数据
