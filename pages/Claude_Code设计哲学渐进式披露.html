Claude Code 设计哲学：渐进式披露（Progressive Disclosure）

来源：微信视频号「Claude Code 的设计哲学：渐进式披露」本地视频（190MB）
综合参考：CSDN 深度文章 + Anthropic 官方「Equipping agents for the real world with Agent Skills」+ 掘金架构演进分析

---

一句话总结

渐进式披露（Progressive Disclosure）是 Claude Code 的核心设计哲学——不一次性把所有信息塞给模型，而是给模型工具让它自己层层深入，借鉴自 UI 设计领域「只在需要时展示复杂信息」的原则。

---

核心三句话本质

  1. **是什么**：渐进式披露是 Claude Code Agent Skills 系统中的上下文加载策略，将技能的**可发现性元数据**与**完整执行指令**分层存储、按需加载。
  2. **解决什么问题**：解决"技能数量 × 完整指令体积"导致的上下文窗口溢出——100 个技能全量加载需 ~500,000 tokens。
  3. **怎么用**：通过 `SKILL.md` 文件分离 YAML Frontmatter（元数据层）和 Markdown 正文（指令层），加外部资源文件（资源层），构成**三层按需加载架构**。

---

演进背景：从"塞信息"到"给工具"

RAG 路线的问题

  • - 索引和设置过程繁琐，不同环境表现不稳定
  • - 上下文是"给" Claude 的，而不是它自己"找"的（被动接受）
  • - 检索质量高度依赖嵌入模型质量，常返回相关但不精确的结果

转折点

"如果 Claude 能在网上搜索信息，为什么不能在代码库里搜索？"

Grep 工具的引入催生了"渐进式披露"理念——给 Claude 工具让它自己决定搜什么、怎么搜、搜到后怎么用。

实际观察发现：智能体被赋予搜索自主权后，表现出惊人的判断力，不会滥用能力把上下文搞爆炸。

渐进式披露成为 Anthropic 添加新功能的首选策略

  • - 与其给 Claude 增加一个新工具，不如给它一个入口让它自己探索更深层资源
  • - 好处：保持工具集精简（避免选择困难）+ 提供功能可扩展性（通过递归引用）
  • - 核心理念：「当智能体足够聪明时，你不需要为它准备好一切，只需要为它打开正确的门」

三层架构实现

Claude Code 最终的解决方案是构建 `claude-code-guide` subagent：


用户问：「如何添加 MCP？」
        │
        ▼ 主 Claude 识别意图
        │
        ▼ 触发 claude-code-guide subagent
        │
        ┌────┴──────────────────────┐
        │ subagent 内部               │
        │ ├─ 按指令搜索文档           │
        │ ├─ 递归读取相关文件         │
        │ └─ 精准整理并返回答案        │
        └───────────────────────────┘
        │
        ▼ 主 Claude 回答用户
        （此过程不污染主上下文）

Agent Skills 的三层披露（官方定义）

层级 内容 加载时机
第一层：元数据 SKILL.md 的 YAML frontmatter（name + description） 启动时全部加载到 system prompt
第二层：核心指令 SKILL.md 的正文 Markdown Claude 认为相关时按需加载
第三层：详细资源 引用的 `reference.md` / `forms.md` 等附属文件 Claude 主动点击进入时加载
Skills 包含的上下文量"实际上是无限的"——因为只有需要时才读。

---

三层核心价值

1. 工程价值：零工具扩展能力

传统方式 渐进式披露
新功能 → 新增工具 新功能 → 写 skill 文件或 subagent
修改代码部署 只需增加文档/说明文件
硬编码进系统 动态可插拔

Claude Code 仅有约 20 个工具，扩展门槛极高。渐进式披露让团队可以不触碰工具数量持续扩展 Claude 的能力边界。

2. 认知价值：减少干扰，聚焦核心

  • - 主 Agent 的上下文保持干净，专注于写代码
  • - 知识型内容按需加载，互不干扰

3. 架构价值：专业分工

  • - 每个 subagent 有专属的搜索指令和返回格式
  • - 比通用 Agent 精准得多

关键洞察

  1. **RAG 地位下降**：从"主角"变成了 Agent 工具箱里的一个工具。这是 Agent 时代的范式转变。
  2. **Skills 是 MCP 之外的另一条扩展之路**：MCP 解决"工具/数据接入"，Skills 解决"知识/方法论接入"。两者正交。
  3. **「克制」是核心**：「给得越多越好」的思维是反直觉的——Claude Code 团队反复实验证明，**最小化初始上下文 + 最大化自主探索能力**才是正确方向。
  4. **Token 消耗优化**：渐进式披露可降低 60%-80% 的 Token 消耗，指令遵循率大幅提升。
