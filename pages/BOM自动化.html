交互式 BOM 表（AI 自动生成 PCB 物料清单）

核心定义

**一句话定位**："交互式 BOM 表"是 AI Agent 时代的物料清单新形态——把传统 Excel BOM 升级为可对话、可追溯、可下单、可版本管理的「活动文档」。KiCad MCP、NextBoard、bom-generator 等多种工具已落地。

核心功能

  • - **自动选型**：从需求描述 → 推荐合适器件 + 国产替代
  • - **Datasheet 自动下载**：避免工程师去官网逐个查
  • - **实时比价**：多分销商（DigiKey / Mouser / LCSC / element14 / JLCPCB / PCBWay）瞬时对比
  • - **兼容性检查**：电压 / 电流 / 封装 / 接口匹配
  • - **库存预警**：避免选停产 / 缺货器件
  • - **下单文件生成**：BOM → 嘉立创 / PCBWay 订单格式

代表工具全景

  1. **KiCad MCP**（最直接）：通过 MCP 协议调用 KiCad，做项目管理、设计分析、网表与 BOM 提取、自动化 DRC 报告。详见 KiCad+AI工作流。
  2. **NextBoard**（最完整方案）：github.com/LeoKemp223/NextBoard——面向硬件方案设计的 AI Agent，自动需求确认 → 芯片选型 → 下载 datasheet → 输出 BOM → 计算价格 → 输出模块原理图 → 整合可落地技术方案。安装：`claude plugin marketplace add LeoKemp223/NextBoard` 或 `--plugin-dir`。
  3. **bom-generator Skill**（轻量级）：触发于"我需要什么部件"，输出含 part numbers / quantities / suppliers 的 BOM，导出 xlsx / csv / md / json。
  4. **Claude Code Bom Skill**（最专业）：调度 DigiKey / Mouser / LCSC / JLCPCB / KiCad 多个子 Skill，BOM 数据存为 KiCad 原理图符号属性（single source of truth）。
  5. **generate-bom Skill**（结构化输出）：输入 components.json，输出 Markdown 表格 + CSV（去重合并），可导入采购 / ERP / CAM。

适用场景

适用 不适用
PCB 设计阶段的器件选型与 BOM 初稿 纯机械 / 无源结构件清单（仍需人工）

设计 → 下单一体化（BOM→订单） 已固化的量产物料（ERP 主数据为准）
团队可复用 Skill 沉淀 极简一次性的小样机

团队评估

交互式 BOM 已从"概念"进入"日常可用"：NextBoard 等工具覆盖"需求 → 选型 → 下载 datasheet → 输出 BOM → 输出原理图 → 计算价格 → 整合技术方案"完整链路；多 Skill 协同是 BOM 场景的标配。关键工程原则是 **Single Source of Truth**——所有 BOM 操作基于 KiCad 符号属性，避免数据分散在 Excel/Word/邮件。

关键洞察

  1. **BOM 的范式转移**：从"死的 Excel"到"活的数据资产"——AI 时代的 BOM 与 KiCad 工程文件、供应链、下单系统深度集成。
  2. **"硬件方案设计 Agent"已成型**：NextBoard 不是简单生成 BOM，而是覆盖完整链路。
  3. **多 Skill 协同是 BOM 场景的标配**：Bom Skill 调度多个子 Skill，按需组合。
  4. **「Single Source of Truth」是 BOM 数据管理关键**：所有操作基于 KiCad 符号属性。
