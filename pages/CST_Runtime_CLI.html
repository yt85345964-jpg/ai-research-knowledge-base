CST Runtime CLI（CST MCP + CST Simulation Workflow Skill）

核心定义

**一句话定位**：CST Runtime CLI 是 CST Studio Suite 的 MCP Server——让 AI Agent（Codex、Claude Code 等）通过 MCP 协议直接控制 CST 完成电磁仿真任务，从自然语言需求到 S 参数 / 远场 / Touchstone 文件全自动导出。

为什么重要

纯靠人工驱动电磁仿真软件，建模→求解→导出→对比链路长、迭代慢。AI Agent 接入后把"自然语言任务"直接变成可执行仿真闭环，价值体现在：

AI Agent 在仿真中的价值

  1. **自然语言 → 仿真任务**：用户说"帮我优化 77GHz 雷达的 S11 到 -25dB 以下"，Agent 自动调用 CST 工具链，扫描参数、跑多次仿真、对比结果。
  2. **参数扫描自动化**：g 参数从 20 到 30，步进 0.5（共 21 次仿真），AI 自动编排任务队列、并行执行、收集结果。
  3. **优化闭环**：`cst-runtime-optimization` skill 专门做 S11 优化，完整流程：定义目标 → 扫描参数 → 提取 S 参数 → 评估 → 调整 → 重跑。
  4. **结果可视化**：Agent 自动生成 HTML 对比页（含 S 参数图、远场方向图）。

关键洞察

  1. **AI Agent + 仿真软件 = 工程师的"AI 副驾"**：自然语言任务 → 自动建模 → 自动仿真 → 自动导出 → 自动对比——传统需要数天的仿真优化工作可压缩到数小时。
  2. **MCP 是仿真软件 AI 化的标准协议**：CST Runtime CLI 是 MCP 在 EDA 仿真领域的典型应用，未来 HFSS / ADS / Cadence Virtuoso 都可能采用类似模式。
  3. **毫米波雷达仿真的核心难点在「场景复杂」**：天线罩 + 保险杠 + 多径 + 多目标——AI 价值在于自动编排这些多物理场协同仿真。
  4. **"双求解器交叉验证"是工程最佳实践**：T 求解器 + F 求解器对比，是判断仿真准确性的"金标准"。AI Agent 可以自动完成这个对比。

视频实操案例

任务：汽车 77GHz 毫米波雷达天线罩的 1×4 微带贴片阵列基线仿真。

Agent 执行流程：

  1. 接收自然语言任务（Codex）
  2. 调用 CST MCP 工具启动 CST Studio Suite
  3. 调用 CST Simulation Workflow Skill 编排仿真步骤
  4. 执行 1×4 微带贴片阵列建模
  5. 求解器运行（结果：79.63GHz 附近匹配较好，77GHz 处仍待优化）
  6. 导出产物：CST 项目文件、Touchstone 文件（`.sNp`）、S 参数 CSV、S 参数图、求解器日志

求解器选择：

求解器 适用场景
时域（T-solver） 宽带分析、非线性器件
频域（F-solver） 高 Q 值谐振结构
积分方程法 辐射问题、开放边界
多层快速多极子 超电大尺寸问题
TLM 全波 高精度（资源密集）
A-solver NFS + Radome 组合仿真

雷达罩建模技巧（77GHz 保险杠）：

  • - 多层薄壁结构用 `Thinpanel` 建模（避免网格爆炸）
  • - 每层定义材料参数 → 设置层叠列表 → 3D 薄层等效
  • - WCS 局部坐标系指示层叠方向

双求解器交叉验证：

一定要用不同的解算器来做对比——如果不同算法都能拿到相同的结果，才能说明模拟是准确的。
  • - Boundary reflection level ≤ 1e-6
  • - F 求解器：精度 1e-6 起步，勾选网格自适应加密
  • - T 求解器：boundary 与 F 相同，重点在 local mesh 加密
