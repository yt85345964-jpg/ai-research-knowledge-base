Obsidian+AI工作台

核心定义

**一句话定位**：以 Obsidian 作为本地 Markdown 知识库，接入 AI Agent（Codex / Claude Code / WorkBuddy 等）实现自动化知识管理的本地工作台范式。

核心功能

目标

解决两个核心问题：

  1. **知识沉淀**：将碎片化的AI研究资料持续编译为结构化的 Wiki 知识库
  2. **隐私可控**：所有数据在本地，不依赖云端平台

核心模式


[原始资料] → [AI Agent 处理] → [Obsidian Markdown 文件] → [知识图谱/索引]
   ↑                               ↓
   └────── AI 发现缺口 ←──── 人类审核与反馈

步骤

  1. **知识摄入**：从各种来源收集资料（微信文章、PDF、视频链接等）；AI Agent 读取并提取结构化信息；按模板生成素材笔记
  2. **知识编译**：AI Agent 分析素材笔记，发现主题和联系；创建/更新 Wiki 页面，建立双向链接；检测重复、矛盾、过时内容
  3. **结构维护**：更新索引.md、日志.md；维护跨页面的 wikilinks；清理孤立页面和失效链接
  4. **报告生成**：从知识库提取数据生成周报；统计知识增长、发现缺口；输出团队可读的摘要

与我们的知识库

本团队的知识库本质上就是一个 Obsidian+AI 工作台的实例：

  • - **知识库载体**：Obsidian Vault (`E:\AI-团队知识库`)
  • - **AI Agent**：WorkBuddy（执行同步、编译、报告）
  • - **采集入口**：IMA 知识库「杨添的知识库」
  • - **编译规范**：AGENTS.md（三层架构：原始资料→知识库→The schema）
  • - **调度方式**：WorkBuddy 自动化（每日同步编译、每周报告）
