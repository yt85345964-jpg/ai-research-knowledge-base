AI Agent 8 大主流架构

核心定义

**一句话定位**：把当前最常见的 8 种 AI Agent 构建范式（ReAct / Plan-and-Execute / Multi-Agent / Reflective / Tool-Augmented / Memory-Augmented / RAG / Autonomous Loop）各自用「流程图 + 优点 + 缺点」讲清，构成一张 Agent 架构入门知识图谱。

为什么重要

单个 Agent 的能力受限于上下文窗口与工具边界，复杂任务往往需要特定的架构范式来补足。理解这 8 种范式，是设计、选型、评估 Agent 系统的基础——不存在「银弹架构」，只有针对「成本 / 可靠性 / 复杂度」三者的不同取舍。

对我们团队的启示：这份图谱可以直接作为我们 AI 知识库「编译流程」与「专家团编排」的选型参照——WorkBuddy 专家团偏 Multi-Agent + Tool/LLM 协同，OpenClaw 偏 Harness 执行壳 + 工具接入（MCP），都在这 8 类里有对应位置。

关键机制

8 大架构全景对比

# 架构 一句话定位 核心思路 优点 缺点 自主度
01 **ReAct** 推理与行动协同 想→做→看反馈→再想→再做，循环到完事 简单、啥都能套 来回调 LLM，费钱又慢 ★
02 **Plan-and-Execute** 规划与执行分离 先规划（拆任务、排顺序），再执行（逐个干、验证），搞砸了退回重来 复杂任务好用、可回滚 规划不一定准；小任务太重 ★★
03 **Multi-Agent** 多智能体协作 一个协调器分配，多个 specialist（搜索/写作/编程/质检）共享知识库、最后汇总 效率高、专业分工、像真人团队 协调成本高、易「打架」 ★★
04 **Reflective Agent** 反思型智能体 生成→反思→找错→改→检查，不过再来（最多 3–5 轮） 输出质量高、自我进化 烧钱之王，每轮都调一次 LLM ★★
05 **Tool-Augmented** 工具增强型 LLM 自维护工具列表，自己决定调哪个、拿结果整合回复 从嘴炮变实干、现代 Agent 标配 工具烂的话啥都白搭 ★★
06 **Memory-Augmented** 记忆增强型 四级记忆：当前会话→短期→长期→向量数据库，系统自动搬运清理 不会聊两句就失忆、适合长期交互 管理复杂、存储蹭蹭涨 ★★
07 **RAG Agent** 检索增强生成 查询向量化→搜数据库→排序取最优→拼上下文→LLM 生成 减幻觉、可溯源、知识实时更新 搜得不好就答得不好、得维护知识库 ★★
08 **Autonomous Loop** 自主智能体循环 目标→自动拆任务→排优先级→执行→评估→记下来→追踪进度→循环 真正自动驾驶、适合跑批 可能跑偏、得有人盯着 ★★★

各架构流程要点

01 ReAct（最基础的起点）

`用户输入 → 上下文 → 推理 → 决定 → 行动 → 执行 → 观察 → 反馈 → 环境与工具 → 最终输出`，在「推理」与「行动」间循环。所有其它架构都建立在这个循环之上。

02 Plan-and-Execute

`用户目标 → 任务规划器（子任务分解 + 依赖分析）→ 有序子任务列表 → 执行器（工具调用）→ 结果验证 → 聚合结果`；失败则重规划。把「想」和「做」两阶段切开，便于回滚。

03 Multi-Agent

`用户请求 → 协调器(Orchestrator) → 搜索/写作/编程/质检 Agent → 共享知识库 → 聚合响应`。核心是「专业分工 + 共享状态 + 协调器汇总」，与 多Agent协作模式 一致。

04 Reflective Agent

`用户请求 → 初始化自我模型 → 自我反思 → 错误分析 →（未通过）改进策略 → 调整检查 →（通过）最终输出`，最多 3–5 轮。「自己当自己的质检员」，本质是反思式提示（关联 Agent提示词设计模式）。

05 Tool-Augmented

`LLM Core + Planning&Reasoning` 外接 浏览器/搜索/知识库/代码/分析/自定义 工具。核心是「给大脑配上手脚」，工具接入范式见 MCP 与 Harness(智能体执行壳)。

06 Memory-Augmented

`用户交互 → LLM 处理器 → 感官缓冲 → 短期记忆 → 长期记忆 → 外部存储(Vector DB)`。四级记忆在层级间自动搬运清理，解决「聊两句就失忆」。

07 RAG Agent

`用户查询 → 意图理解 → 查询改写 → 查询向量化 → 检索(Vector KB, 混合检索) → 结果排序(Rerank) → 上下文整合 → LLM 生成 → 输出答案`。先查资料再回答，详见 RAG（检索增强生成）。

08 Autonomous Loop

`长期目标 → 目标分解 →（经验记忆 + 工具系统 + 外部感知）→ 任务优先级队列 → 执行 → 结果评估 → 记忆更新 → 进度追踪 → 下一个任务`，持续循环。自主度最高，但需人监督防跑偏。

关键设计判断

  • - **Tool / Memory / RAG 不是「独立第 5–7 种架构」，而是给 Agent 补「手脚 / 记性 / 外脑」的能力模块**，可叠加在 ReAct、Multi-Agent、Autonomous Loop 之上（例如 Multi-Agent 的每个 Agent 都可带 Tool / RAG / Memory）。
  • - **自主度递增轴**：ReAct（手动机）→ Plan-and-Execute → Multi-Agent / Reflective → Tool/Memory/RAG（补能力）→ Autonomous Loop（全自动）。越往右人类介入越少，但「跑偏 / 烧钱」风险越高。
  • - **选型本质是「成本 × 可靠性 × 复杂度」的取舍**：简单任务用 ReAct；复杂可回滚用 Plan-and-Execute；专业分工用 Multi-Agent；重质量用 Reflective；要干活用 Tool；长期交互用 Memory；要准确可溯源用 RAG；跑批自动化用 Autonomous Loop。
