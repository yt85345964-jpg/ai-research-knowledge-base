一句话总结

Claude（Code）已可深度介入软硬件研发全流程——从嘉立创 EDA 自动画原理图、到嵌入式 MCU 远程调试（AI 调试器 AixProbe），再延伸到平衡车/自动驾驶等具体产品形态，AI 正在成为"看得见硬件、动得了硬件"的研发副驾。

---

三大应用场景

1. 嘉立创 EDA + Claude Code：自动画原理图

实施路径（已成熟）：


# Step 1：安装 easyeda-api skill
claude mcp add --transport stdio kicad ...
# 或
https://github.com/easyeda/easyeda-api-skill  帮我安装这个 skill

# Step 2：嘉立创 EDA 侧安装扩展
# 高级 → 扩展管理器 → 搜 "run" → 安装 → 勾选三个选项

# Step 3：连接
claude code
> EDA 启动！
# → 建立 WebSocket 连接

价值：

  • - 减少重复性手动操作，缩短设计周期
  • - 一致性保证，避免人为错误
  • - 知识沉淀：设计经验 → 可复用自动化脚本
  • - 让工程师专注创造性设计

2. AI 调试器 AixProbe：首个开源 AI 远程调试器

视频未直接展示 AixProbe，但"嵌入式"主题是核心之一，资料来自同主题的《首个AI远程调试器》头条文章。

核心：T113-S3 微型网关设备 + 内置 OpenOCD 引擎 + WiFi 6 + JTAG/SWD 协议 + MCP 服务接口。

架构特点：

  • - AI 能直接"操作"硬件（不仅看代码）
  • - 配套 Skill 技能包涵盖日常工作场景
  • - 两种配置方式：CLI 命令行（嵌入式老炮） + GUI 图形界面（小白）
  • - 上位机自动切换 ssh / adb，电脑端双击单文件 exe 即可运行

核心硬件（基于嘉立创 EDA 设计的四层板）：

模块 型号
主控 T113-S3
模拟开关 CH442Q
DC-DC SY8113BADC
LDO XC6206P252MR
NAND GD5F1GQ4UBYIGR
WiFi/蓝牙 rtl8723bs
USB 转换 CH347F

3. 平衡车 / 自动驾驶：嘉立创 EDA 实战跳板

为什么平衡车是好项目：

  • - 涉及电源管理（锂电池 + DC-DC + LDO）+ MCU（GD32F303C8T6）+ 运动传感器（陀螺仪）+ 执行机构（TB6612 电机驱动）+ 无线通信（蓝牙）+ 传感器接口（编码器、红外）
  • - 几乎涵盖所有嵌入式硬件核心模块
  • - **对硬件可靠性要求极高**：动态系统，任何毛刺都会"翻车"

AI 辅助价值：

  • - AI 可参与电源完整性、信号完整性、EMC 设计的早期评估
  • - 自动检测"未连接引脚"、"相似器件推荐"、"检查连接关系"
  • - 平衡车的"翻车压力"对硬件工程是绝佳实战场景

关键洞察

  1. **AI 硬件研发范式已成型**：从「画板」（EDA 自动化）→「调试」（JTAG + MCP）→「产品」（平衡车/自动驾驶），全链条都有 AI 切入点。
  2. **"AI 看硬件、动硬件"是 2026 的关键突破**：过去 AI 只能看代码、猜硬件 Bug；现在 AI 能直接操作硬件读寄存器、烧录、单步执行。
  3. **国产 EDA 正在主动拥抱 AI**：嘉立创 EDA 推出 Run API Gateway、推出 easyeda-api Skill、推出 Agent Connector 三套方案——这是行业级共识。
  4. **平衡车是 AI 嵌入式练手神器**：兼具挑战性、模块丰富度、社区资源，是验证 AI 全栈辅助的最佳载体。
