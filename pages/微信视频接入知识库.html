微信视频接入知识库

核心定义

**一句话定位**：微信群 / 视频号里看到值得留存的视频，只需「在微信里点开看一眼」，视频即自动落盘到知识库、自动 ASR 转录、自动生成素材笔记，省去手动下载 + 上传。

为什么重要

视频是高质量 AI 资讯的重要载体（技术拆解、产品发布、访谈复盘），但微信的视频散落在群聊 / 视频号，传统方式需手动下载再上传到知识库，链路长、易遗漏。本方法把"查看即捕获"做成端到端管道，让视频内容像文章一样自动进入知识库摄入流水线。

关键机制

机制概述

端到端管道


你在微信群 / 视频号里【查看或播放】想留存的视频
   │  WeChat 自动经本地代理(127.0.0.1:2023)拉取视频
   ▼
wx_video_download 拦截视频 URL，用 Gopeed 下载到（已配置）：
    E:/AI-团队知识库/原始资料/待处理/<视频>.mp4
   ▼
watch_inbox.py 检测到新视频（写入稳定 + 可被读取，跳过 .part 临时文件）
   ▼

   ├─ asr_video.py：faster-whisper 音频转录（中文）
   ├─ 生成素材笔记 → 知识库/素材笔记/待分类/<标题>.md
   └─ 原始视频归档 → 原始资料/<标题>.mp4
   ▼
WorkBuddy 编译：素材笔记 → Wiki 页面（人工或自动化触发）

两种接入方式对比

方式 覆盖来源 是否需要转发 合规 推荐度
**A. wx_video_download 代理捕获（本方案主推）** 群视频 + 视频号 + 私聊 不需要，点开看即可 中（装本地代理+根证书，非微信官方） ★★★ 覆盖群视频
B. OpenClaw / 小龙虾 单聊转发 仅单聊文件 需要长按转发给 ClawBot 高（微信官方 iLink） ★★ 仅单聊
关键事实：**微信官方 iLink / ClawBot 机器人不能进群监听，只能收单聊文件**，所以「机器人自动爬群视频」在官方框架下做不到。但 wx_video_download 走的是另一条路——**代理拦截微信自身的下载流量**，因此能抓到群视频和视频号，前提只是「你在微信里查看过它」。

#### 主方案：wx_video_download 按需捕获（端到端管道配置）

一次性配置（已为你改好）

编辑下载器配置 `D:\软件\微信视频下载\wx_video_download_v260614_windows_x86_64\config.yaml`：


download:
  dir: "E:/AI-团队知识库/原始资料/待处理"   # 原值 "%UserDownloads%"，已改为直接落盘待处理
已备份原配置为同目录 `config.yaml.bak`。改回 `%UserDownloads%` 即可恢复默认。

日常使用（按需启动）

启动：在桌面为 `wx_capture.py start` 建一个快捷方式（目标填 `C:\Users\tian\.workbuddy\binaries\python\versions\3.13.12\python.exe "E:\AI-团队知识库\脚本\wx_capture.py" start`），双击即可——脚本自提权弹 UAC，点「是」开捕获窗口；想省去 UAC 弹窗可在快捷方式属性勾「以管理员运行」。

捕获窗口打开后，去微信里查看/播放想留存的视频 —— 它们会自动出现在 `原始资料/待处理/`。

捕获完毕，关闭方式：直接在那个启动窗口按 Ctrl+C。

⚠️ 关闭务必用窗口内的 **Ctrl+C**（脚本会先恢复系统代理、再杀掉下载器、干净退出，最稳妥）。不要对 AI 说「关闭代理」——WorkBuddy 对话触发弹不出可见窗口，无法让你 Ctrl+C（详见下方 WorkBuddy 触发限制）。仅当启动窗口异常退出、残留进程时，双击桌面「微信视频捕获-停止」快捷方式（`python wx_capture.py stop`）兜底。

查看状态（管理员 PowerShell）：`python 脚本/wx_capture.py status`

知识库侧：常驻 watch 脚本

脚本 `脚本/watch_inbox.py` 常驻运行，自动把待处理目录里的新视频送去处理。


# 常驻（推荐用 Windows 任务计划程序或常开终端）
python 脚本/watch_inbox.py
# 调试：只处理当前已有的视频后退出
python 脚本/watch_inbox.py --once
# 调 ASR 模型（默认 small，CPU int8；medium 更准但更慢）
KB_ASR_MODEL=medium python 脚本/watch_inbox.py

防半文件保护：watch_inbox 要求文件「大小连续 5 秒不变」且「能被以读模式打开」（下载器仍独占写入时会跳过），避免 ffmpeg 读到不全的视频；`.part / .crdownload / .tmp / .download / .!qb` 等临时文件直接忽略。

#### OpenClaw / 小龙虾 触发方案（计划，待你确认后执行）

推荐 UX：用聊天一句话「开始抓视频 / 停止抓视频」让小龙虾开/关捕获窗口。

为什么这个方式更好（论证）

  • - **真正覆盖群视频**：小龙虾只负责"开/关捕获窗口"指令；视频由 wx_video_download 的代理在你**微信里查看**时抓取。
  • - **比「转发给 ClawBot」方案更强**：旧方案受限于单聊、且常遇"视频号卡片只是链接"。触发方案完全不涉及"接收文件"，只跑一条命令。
  • - **保留你要的"小龙虾感"**：仍用聊天驱动。
  • - **合规**：触发是单聊指令，属 iLink 允许范围。

可行性前置条件（必须逐条满足）

  1. **OpenClaw 拥有本地命令执行能力**：需 terminal / command 工具。
  2. **同机执行**：OpenClaw 执行环境须与本机微信同机。
  3. **Python 可用**：`python` 在 PATH，或指令写绝对路径。
  4. **管理员权限（关键）**：安装根证书需提权；`wx_capture.py start` 已内置 UAC 自提权。

4. 不要对视频做任何其他处理。

关闭方式（无需对 AI 说话）：
- 用户在启动窗口按 **Ctrl+C** 即可。

  python "E:/AI-团队知识库/脚本/wx_capture.py" stop   （兜底清理）

推荐配套

  • - `watch_inbox.py` 保持常驻（Windows 任务计划程序开机自启）。
  • - 若不想常驻，就在 **Ctrl+C 关闭后**让助手跑一次 `watch_inbox.py --once`。

风险与边界

  • - **权限边界**：小龙虾获得本机命令执行权 = 等同于给了 AI shell 权限。仅单聊、仅你本人可发指令。
  • - **忘记关**：代理与根证书持续生效。已用两道护栏消解：① 启动输出软提醒；② 默认 30 分钟看门狗前台超时自动恢复代理并退出（`start --watchdog 0` 可关）。
  • - **自然语言歧义**：明确触发词，避免日常聊天误触发。
  • - **证书/代理未恢复**：Ctrl+C / 看门狗超时自动恢复；若启动窗口异常崩溃残留进程，运行 `stop` 兜底。

远程控制可行性（计划，待你确认）

结论：**「远程发指令开关」完全可行；但「人不在场时全自动抓到群视频」存在断点。**
  • - 哪部分可行（✅）：OpenClaw 本机常驻 + iLink 接入个人微信 → 外面用手机微信发「开始抓视频」，本机即时执行 `start`。前提：command 权限、同机、管理员、电脑开机不休眠、微信 PC 端登录。
  • - 隐藏断点（⚠️）：`wx_video_download` 是**代理拦截**，依赖**本机微信真正加载/播放视频**时才发 HTTPS 请求被拦截。手机微信点开 → 流量走手机网络，不经本机代理 → 抓不到；iLink 进不了群监听 → 连"群里有新视频"都不知道。
  • - 补全方案（让远程也能全自动）：
方案 做法 可行性 代价
**A. UI 自动化补点**（推荐补） 小龙虾触发 `start` 后，再跑 pyautogui 脚本自动激活微信 PC 端 → 打开指定群 → 点播放 可行（模拟鼠标，不逆向微信） 需电脑不锁屏、微信前台；界面改版会脆
**B. 视频号自动流** 保持视频号 tab 打开，利用自动播放让流经内容被抓 部分可行 不可控、不针对群
**C. 退一步半自动** 远程只负责「开窗口预热」；真正抓取回电脑前看一眼 100% 可行 不是无人值守
推荐组合：**2.3 指令 + 2.7.3-A 自动点开脚本**。注意 A 只能抓「事先指定群」里**已存在**的视频。

远程场景额外检查清单

  • - [ ] 电脑「永不休眠」，微信 PC 端保持登录
  • - [ ] OpenClaw 常驻（开机自启）
  • - [ ] 若用方案 A：微信 PC 端登录态且不锁屏；自动点开脚本已配置目标群
  • - [ ] 网络长期可用（电脑端）

「转发给龙虾 → 触发代理」设想澄清（计划）

用户设想：转发视频给小龙虾 → 自动开代理 → 下载 → 关代理。探查后结论：**这条链技术上拼不通**，因为「转发」和「代理拦截」是两套互斥路径。
  • - 探查到的事实（来自 `wx_video_download --help`）：工具本意是启动后对网络请求代理，在视频号详情页注入下载按钮——触发点是**微信内 UI**；存在子命令 `download --url [--key <密钥>] [--filename <名>]`：理论上可不开代理精准下载，但视频号加密需 `--key`，直链+key 只能从微信内部获取。
  • - 核心矛盾：
  • - 情况 A｜转发的普通视频文件：iLink 单聊能收到 → OpenClaw 直接写盘，wx_video_download 多余。最干净。
  • - 情况 B｜转发的视频号卡片：开代理后真正能下载的前提是本机微信重新加载该视频。无法精准只抓转发那条。
  • - 结论：`download --url` 虽支持精准下载，但直链+key 来源（微信内部）缺一段桥。
  • - 给用户的修正建议：
视频类型 推荐方式 是否用代理
群直接发的视频文件 **转发给小龙虾 → OpenClaw 直接收文件写盘** → watch_inbox 处理 ❌ 不用
视频号卡片 **在微信里打开看** → 代理拦截抓 ✅ 用，看完 Ctrl+C 关闭

「对话开关 + 自动关代理 + 进展汇报」方案评估（计划）

用户最终方案（2026-07-11）：对话「下载视频」→ 开代理；用户手动看视频；下载完自动关代理；小龙虾汇报；路径已在库内。
  • - 总体结论：**可行，且比 2.3 更闭环**——顺带解决「忘记 stop」风险。三项要件：① 对话 start ✅；② 用户手动看 ✅；③ 下载完自动 stop ⚠️ 需补 watcher；④ 汇报 ✅；⑤ 路径在库内 ✅。
  • - 核心难点：如何判定「下载完成」——需自写 watcher（基于文件系统 watch 最稳，复用 watch_inbox 稳定判定）。状态机：IDLE → 待处理出现新 mp4 → DOWNLOADING → 稳定 → SILENT（静默 N=30s）→ 期满 stop。
  • - 实现封装（计划，不执行）：新增 `wx_capture.py capture-session` 子命令（前台阻塞，下载完自动 stop 并返回）。
  • - 仍需注意：捕获范围——代理开着会抓"你看过的所有微信视频"；前置闸门同 2.2；本机微信需加载。
  • - 对比旧方案：
维度 对话开 + Ctrl+C 关 对话开 + 自动关
启动代理 对话触发 对话触发
关闭代理 **窗口内 Ctrl+C** **下载完自动关**
忘记关风险 低（看门狗兜底） 消除
多视频灵活性 **高** 需静默期兜底
代理残留风险 **低** 中

变体：对话开、手动 Ctrl+C 关（用户 2026-07-13 最终决定）

**启动仍用对话指令**（`start` 自动提权 + 前台守护）；**关闭不再用对话指令**，改为启动窗口直接按 **Ctrl+C**。
  • - 本质：`wx_capture.py start` 现在是**前台守护**——脚本前台运行、把下载器当子进程拉起并捕获 Ctrl+C，先恢复系统代理 → 再杀下载器 → 干净退出。
  • - 优劣（相对自动关）：工程最简单、关闭最稳、不会残留代理；多视频最灵活。
  • - 结论与建议：采用 2.10 作为当前落地形态；"忘记关"已被两道护栏消解（软提醒 + 30 分钟看门狗）；若日后想要"下完自动关"，再升级到 2.9。
  • - 对话/AI 触发指令（可直接粘贴）：

#### 替代方案：直接用 WorkBuddy 助理触发（无需 OpenClaw）

结论：可以，且比 OpenClaw 路径更简单——前提是你接受触发入口在 WorkBuddy 而非微信。
  • - 概念澄清：「小龙虾」= OpenClaw（开源 AI Agent 框架）；WorkBuddy = 当前这个 AI 助理（与 OpenClaw 是两个不同东西）。
  • - WorkBuddy 直接做：理论可行，实测受沙箱限制。⚠️ **实测限制（2026-07-13 验证）**：WorkBuddy 的 Bash 工具运行在**受限/隔离沙箱**中，弹不出任何 GUI 子进程（UAC 提权窗、独立控制台窗口均被吞）。因此「对话触发 start → 弹窗口 → 你按 Ctrl+C 关闭」在当前环境物理上不成立。
