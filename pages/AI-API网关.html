AI API 网关

一句话定位

AI API 网关是企业内部统一管理和调度 AI 模型 API 调用的中间层，提供令牌管理、流量控制、模型路由等能力，降低团队接入 AI 的门槛。

核心能力

统一令牌管理

  • - 管理员创建令牌，绑定可用模型列表和额度限制
  • - 员工只需复制密钥即可使用，无需各自申请模型厂商的 API Key

流量控制与治理

  • - 统一的调用额度管理，避免个人超量
  • - 可设置 IP 限制、过期时间等安全策略
  • - 集中监控和日志审计

模型路由

  • - 向员工暴露统一的兼容接口（通常为 OpenAI 兼容协议）
  • - 后端可灵活切换实际模型厂商和版本
  • - 对接多个模型（DeepSeek、Claude、GLM 等）

企业价值

场景 无网关 有网关
员工接入 每人单独申请 API Key 管理员一次配置，全员复用令牌
成本控制 难以追踪个人用量 集中额度管理和日志审计
模型切换 每人修改配置 网关层统一路由，用户无感
安全合规 密钥分散管理 集中管控，可设置 IP 白名单

已知案例

单一来源：素材笔记/工作流/2026-07-02_Workbuddy配置公司中转站操作手册
华诺星空公司部署了 AI 网关（aigateway.novasky.cn），支持 deepseek-v4-pro 模型。员工通过 OA 系统的"AI 网关"入口获取令牌，然后在 WorkBuddy 中配置自定义模型即可使用。

产品关联

  • - WorkBuddy — 支持通过 OpenAI 兼容协议接入自定义模型，AI 网关是其企业部署场景的标准接入方式
  • - Codex — 同为 AI Agent 平台，理论上也可通过类似方式接入企业网关（待验证）
