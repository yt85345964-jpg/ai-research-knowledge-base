2 步教会你用 AI 操控嘉立创 EDA 画 PCB

来源：微信视频号「AI 操控嘉立创 EDA 画 PCB」本地视频（68MB）
时间：2026-07-11 入库
视频信息密度较低，主要演示价值；本素材笔记综合嘉立创 EDA 官方 + easyeda-agent/easyeda-api Skill + Run API Gateway 三套社区方案整理

---

一句话总结

嘉立创 EDA 专业版已可通过官方扩展 / 第三方 Skill / Agent Connector 三种路径实现"AI 直接操控编辑器画原理图与 PCB"，目前已证明路径可行但效果一般——复杂场景仍需工程师介入。

---

三套方案对比

方案一：社区 Skill（easyeda-api / easyeda-agent）⭐⭐⭐⭐

架构：Skill 描述工作流 → Go CLI/Daemon (端口 49620-49629) → WebSocket → .eext Connector → eda.* API

安装步骤：


# 1. 安装 easyeda-agent
curl -fsSL https://raw.githubusercontent.com/zhoushoujianwork/easyeda-agent/main/install.sh | sh

# 2. 在嘉立创 EDA 扩展管理器安装 EasyEDA Agent Connector 扩展
#    仓库: github.com/zhoushoujianwork/easyeda-agent (Apache-2.0)

# 3. 启动守护进程
easyeda daemon

# 4. 验证
easyeda health
# → daemon: online, connector: connected

AI 控制命令示例（与 Hermes / Claude Code / Codex 配合）：


# 搜索器件
easyeda sch search --keyword STM32H753

# 放置芯片
easyeda sch place --lib-uuid xxx --designator U1 --x 3000 --y 3000

# 单步连引脚
easyeda sch connect-pin --designator U1 --pin PA9 --net USART1_TX

# 电源/地标
easyeda sch power-flag --net VDD_3V3
easyeda sch ground-flag --net GND

# DRC + 导出
easyeda sch drc
easyeda sch export netlist
easyeda sch export bom --format xlsx

方案二：官方 eext-run-api-gateway ⭐⭐⭐⭐

架构：AI 编程工具 (Cursor/Claude Code/Hermes) ← WebSocket → eext-run-api-gateway → LCEDA 编辑器

  • - 在嘉立创 EDA Pro → 扩展 → 扩展广场 搜索 `eext-run-api-gateway` 安装
  • - 网关默认端口 9527
  • - 通过 WebSocket 调用 LCEDA 99% 功能

Python 客户端示例：


import asyncio, websockets, json

async def lceda(action, params):
    # ... 连接到 ws://localhost:9527 并发送 typed action
    pass

# DRC 检查
await lceda("run_drc", {"type": "sch"})
# 导出 BOM
await lceda("export_bom", {"format": "csv"})

方案三：自定义扩展文件桥接 ⭐⭐

  • - 传统方式：Hermes Agent 将网表写入 JSON 文件
  • - LCEDA 扩展轮询读取后执行
  • - 通用性差，仅作过渡方案

实际能力边界

视频作者实测后评价："虽然效果一般般，但是已经证明了这个路径的可能"。

  • - ✅ 适用：简单原理图模块搭建、基础器件放置、单步连线、BOM/Netlist 导出
  • - ❌ 不足：复杂 PCB 布局、高速信号规则布线、多层板电源平面优化
  • - ⚠️ 仍需工程师：DRC 误报甄别、关键参数（阻抗/差分/等长）调整

---

关键洞察

  1. **AI 操控 EDA 已"证明可行"**：从代码/自然语言到真实编辑器里的 typed action，已经走过 0→1 阶段。瓶颈在 1→100 的工程化细节。
  2. **三方案生态井喷**：官方 gateway + 社区 Skill + Agent Connector，说明这是行业级共识，嘉立创 EDA 正主动接入 AI Agent 生态。
  3. **计算密集型场景**：AI 协作需要 32GB+ 内存，复杂工程 64GB+；Node.js Bridge Server 默认堆内存 1.4GB 不足，需要 `--max-old-space-size=4096`。
